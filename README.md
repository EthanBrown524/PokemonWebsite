# Pokemon Battle Simulator

Small Tkinter project that loads Pokemon data from CSV files, runs a random auto battle, and now includes a basic machine-learning matchup predictor.

## Run

```powershell
cd "D:\ClassProjects\ActualProj4 - Copy"
python .\src\main_game.py
```

The battle log will show the machine-learning prediction for the current matchup if a trained model exists.

## Train The Predictor

```powershell
cd "D:\ClassProjects\ActualProj4 - Copy"
python .\src\prediction.py --train
```

This builds `data/battle_model.json` by:

- generating random Pokemon matchups from the CSV data
- simulating those battles with the current battle engine
- training a small logistic model on matchup features like stat differences, move quality, and type pressure

## Predict One Matchup From The Terminal

```powershell
cd "D:\ClassProjects\ActualProj4 - Copy"
python .\src\prediction.py --matchup Pikachu Charizard
```

## How It Works

- The app picks both Pokemon randomly.
- Press `Random Matchup` to roll a new fight.
- Press `Start Battle` to watch the battle play out automatically.

## Project Notes

- Sprites are loaded from the URLs in `data/pokemon.csv` and cached in `assets/sprites/cache/`.
- The predictor learns from self-play data generated by this project, so it is a simple starter model rather than a competitive-grade Pokemon AI.

## ML Files

- `src/prediction_features.py` builds the numeric matchup features the model learns from.
- `src/prediction_model.py` contains the small no-dependency logistic regression model, training loop, and save/load helpers.
- `src/prediction.py` handles training data generation, battle simulation for labels, model training, and command-line prediction.
- `data/battle_model.json` stores the trained weights after you run the training command.
